# AWS Audit Manager — Best Practices

## Common scenarios
- Continuously collecting evidence for compliance frameworks (SOC 2, PCI DSS, CIS)        → Security, Operational Excellence
- Running assessments across a multi-account AWS Organization        → Security, Reliability, Operational Excellence
- Protecting and encrypting collected evidence and assessment reports        → Security
- Searching, filtering, and exporting evidence for auditor review        → Operational Excellence

## 🔒 Security
- **[Identity and access]** Start from the `AWSAuditManagerAdministratorAccess` managed policy for audit owners and scope it down to least privilege over time — audit owners need broad control over assessments, frameworks, and controls, but permissions should still be tailored to your use case. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/security_iam_id-based-policy-examples.html)
- **[Identity and access]** Give delegates only management or read-only access instead of administrator access — delegates only need to review evidence, update control status, and submit control sets for their assigned scope. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/security_iam_service-with-iam.html)
- **[Identity and access]** Use IAM Access Analyzer to validate identity-based policies for Audit Manager — it checks policies against IAM best practices and flags overly permissive grants. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/security_iam_id-based-policy-examples.html)
- **[Identity and access]** Require MFA and add SSL-only conditions to identity-based policies that govern Audit Manager access — this adds defense-in-depth around who can create, modify, or delete audit evidence and reports. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/security_iam_id-based-policy-examples.html)
- **[Encryption]** Encrypt Audit Manager resources (assessments, controls, frameworks, evidence, assessment reports) with a customer managed KMS key instead of the default AWS owned key — this lets you view and manage the key, audit its use via CloudTrail, and control access independently of the S3 bucket that stores reports. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/data-protection.html)
- **[Encryption]** Treat KMS key deletion or disabling as destructive for Audit Manager — once a key used to encrypt Audit Manager resources is deleted or disabled, the associated evidence and reports become permanently unrecoverable. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/data-protection.html)
- **[Assessment reports]** Restrict the S3 bucket used as the assessment report destination with a bucket policy that allows only Audit Manager (and required reviewers) to write and read reports, and scope the matching IAM policy to that specific bucket and KMS key ARNs. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/security_iam_id-based-policy-examples.html)
- **[Cross-service access]** Add `aws:SourceArn` and `aws:SourceAccount` (or `aws:SourceOrgID`/`aws:SourceOrgPaths` for org-wide cases) conditions to resource policies that grant Audit Manager or related service principals access — this prevents the cross-service confused deputy problem where a calling service is coerced into acting on the wrong account's resources. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/cross-service-confused-deputy-prevention.html)
- **[Multi-account]** When designating a delegated administrator for Audit Manager, ensure that account has access to the customer managed KMS key configured for encryption — otherwise the delegated administrator cannot decrypt evidence or reports across member accounts. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/delegated-admin-issues.html)

## 🛡️ Reliability
- **[Multi-account setup]** Enable AWS Organizations with all features turned on before integrating Audit Manager across accounts — this is a prerequisite for consolidating evidence collection and running assessments across your organization. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/setup-recommendations.html)
- **[Multi-account setup]** Designate a delegated administrator account (never the management account) for Audit Manager, and use the same delegated administrator account consistently across every AWS Region where Audit Manager is enabled — mismatched delegates per Region break cross-account evidence consolidation. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/delegated-admin-issues.html)
- **[Evidence sources]** Enable AWS Config, AWS Security Hub, and CloudTrail consistently in every in-scope account and Region — Audit Manager depends on these services to automatically generate compliance check, configuration, and management-event evidence, so gaps in their coverage create gaps in your evidence. [doc](https://aws.amazon.com/audit-manager/faqs/)

## ⚙️ Operational Excellence
- **[Evidence collection]** Prefer the automated evidence sources (AWS Config, Security Hub, CloudTrail, service-to-service API calls) over manual uploads wherever possible, and reserve manual evidence for artifacts that truly can't be automated, such as policy documents or training transcripts — this reduces audit fatigue and keeps evidence current. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/OBP.html)
- **[Evidence review]** Enable Evidence Finder and use its filtering and grouping (by assessment, date range, resource compliance, control, or resource ID) to progressively narrow searches instead of browsing nested evidence folders — this scales review across large assessments and multi-account organizations. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/evidence-finder.html)
- **[Delegation]** Use the delegation workflow to assign specific control sets to subject-matter experts for review rather than routing all evidence through a single reviewer — this distributes audit prep work and keeps reviews aligned with domain expertise. [doc](https://aws.amazon.com/blogs/aws/aws-audit-manager-simplifies-audit-preparation/)
- **[Monitoring]** Create a CloudTrail trail that captures Audit Manager API activity (for example `CreateControl`, `DeleteControl`, `UpdateAssessmentFramework`) across all Regions, and use the CloudTrail `userIdentity` element to confirm whether changes came from expected roles or federated users — this gives an auditable record of who modified assessments, frameworks, and controls. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/logging-using-cloudtrail.html)
- **[Framework selection]** Start from the prebuilt AWS Operational Best Practices framework (a subset of AWS Foundational Security Best Practices controls) to baseline audit readiness before layering on custom or industry-specific frameworks. [doc](https://docs.aws.amazon.com/audit-manager/latest/userguide/OBP.html)
- **[Compensating controls]** Treat Audit Manager evidence as an input to your compliance process, not a substitute for it — the service does not assess compliance on your behalf, so continue engaging qualified third-party assessors for formal certification against your target frameworks. [doc](https://docs.aws.amazon.com/prescriptive-guidance/latest/security-reference-architecture/security-tooling.html)

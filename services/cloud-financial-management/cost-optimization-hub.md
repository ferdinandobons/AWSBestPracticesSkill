# AWS Cost Optimization Hub — Best Practices

## Common scenarios
- Centralizing rightsizing, idle-resource, and commitment (Savings Plans/RI) recommendations across accounts        → Cost Optimization, Operational Excellence
- Benchmarking and tracking organization-wide cost efficiency over time        → Cost Optimization, Operational Excellence
- Prioritizing which recommendations to act on first by estimated savings and effort        → Cost Optimization, Operational Excellence
- Rolling out recommendation visibility across an AWS Organizations account structure        → Security, Operational Excellence

## 🔒 Security
- **[IAM]** Grant only the specific `cost-optimization-hub:*` actions a role needs (e.g. `ListRecommendations`, `GetRecommendation`) rather than attaching `CostOptimizationHubAdminAccess` broadly, reserving admin access for roles that must change enrollment or preferences. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/billing-permissions-ref.html)
- **[IAM]** When opting in only a single account, use the narrower single-account opt-in policy instead of the organization-wide policy, since the latter also grants `organizations:EnableAWSServiceAccess` for trusted access. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-getting-started.html)
- **[Organizations]** Understand that the service-linked role `AWSServiceRoleForCostOptimizationHub` is granted read-only permissions (`organizations:Describe*/List*`, `ce:GetCostAndUsage`, `ce:GetDimensionValues`, `ce:ListCostAllocationTags`) — verify this matches your expectations before enabling trusted access for the organization. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/cost-optimization-hub-SLR.html)
- **[Organizations]** Use a single delegated administrator account to centralize visibility into member-account recommendations instead of distributing admin permissions across multiple accounts, since only one delegated administrator is supported per organization. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-getting-started.html)

## 💰 Cost Optimization
- **[Enrollment]** Enable Cost Optimization Hub for the organization's management account with all member accounts included so recommendations are consolidated across the entire account structure instead of reviewed account-by-account. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-getting-started.html)
- **[Preferences]** Set the savings-estimation preference to incorporate your actual discounts (Reserved Instances, Savings Plans) rather than public pricing, so prioritization reflects real dollar impact instead of list-price estimates. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-preferences.html)
- **[Preferences]** Customize commitment preferences (term length and payment option for Savings Plans/Reserved Instances) to match your organization's actual purchasing posture instead of relying on the default three-year all-upfront assumption. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-savings-opportunities.html)
- **[Prioritization]** Sort and group recommendations by estimated savings and implementation-effort indicator to focus first on high-return, low-effort actions rather than working through recommendations in an arbitrary order. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-prioritize-opportunities.html)
- **[Prioritization]** Filter by account, Region, resource type, or tag to identify which segments of the environment hold the most concentrated savings opportunity before broadly rolling out changes. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-prioritize-opportunities.html)
- **[Deduplication]** Review grouped/related recommendations before acting, since Cost Optimization Hub deduplicates overlapping options (e.g. delete vs. rightsize the same instance) and picks the highest-savings action — implementing one recommendation can reduce or invalidate the savings shown for a related one. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-savings-opportunities.html)
- **[Cost efficiency]** Track the cost efficiency metric (aggregated estimated monthly savings divided by optimizable spend) over time as a benchmark, and set organizational goals against it rather than treating individual recommendations as one-off actions. [doc](https://aws.amazon.com/aws-cost-management/cost-optimization-hub/faqs/)
- **[Data exports]** When also using Data Exports for Cost Optimization Hub, leave "Include all recommendations" unchecked (or set `INCLUDE_ALL_RECOMMENDATIONS` to `FALSE`) to avoid double-counting potential savings across overlapping recommendation types. [doc](https://aws.amazon.com/blogs/aws-cloud-financial-management/generate-your-cost-optimization-reports-with-data-exports-for-cost-optimization-hub/)

## ⚙️ Operational Excellence
- **[Enrollment]** Enable Cost Optimization Hub early and expect up to a 24-hour delay before recommendations first populate, then daily refreshes afterward — plan initial reviews accordingly rather than expecting immediate data. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-getting-started.html)
- **[Workflow]** Use Cost Optimization Hub as the single starting dashboard for cost optimization instead of checking Compute Optimizer, Cost Explorer, and other CFM tools separately, then deep-link into the source service console only for implementation detail. [doc](https://aws.amazon.com/blogs/aws/new-cost-optimization-hub-to-find-all-recommended-actions-in-one-place-for-saving-you-money/)
- **[Analysis]** Use the "Analyze with Amazon Q" option on a selected recommendation for deeper investigation before implementing higher-impact or higher-risk changes. [doc](https://docs.aws.amazon.com/cost-management/latest/userguide/coh-savings-opportunities.html)
- **[Governance]** Periodically revisit Cost Optimization Hub preferences (discount visibility, savings-estimation mode, commitment terms) as team engagement matures, since customizing recommendation preferences correlates with higher, more trusted cost-efficiency outcomes. [doc](https://aws.amazon.com/blogs/aws-cloud-financial-management/the-aws-state-of-cost-efficiency-report/)

# Amazon SageMaker — Best Practices

## Common scenarios
- Training and fine-tuning ML/foundation models at scale        → Reliability, Performance Efficiency, Cost Optimization
- Hosting real-time, serverless, asynchronous, or batch inference endpoints        → Security, Reliability, Cost Optimization
- Notebook-based data science and experimentation (Studio/notebook instances)        → Security, Operational Excellence
- Continuous monitoring and retraining of production models        → Reliability, Operational Excellence

## 🔒 Security
- **[Notebook instances]** Disable root access and scope the execution role to least privilege instead of broad service-level policies (for example replace `s3:*` with specific actions on specific bucket ARNs). — Limits blast radius if a notebook or lifecycle configuration script is compromised. [doc](https://docs.aws.amazon.com/securityhub/latest/userguide/exposure-sagemaker-notebook-instance.html)
- **[Notebook lifecycle configs]** Restrict who can create or modify lifecycle configurations and review CloudWatch Logs (`/aws/sagemaker/NotebookInstances`) regularly. — Lifecycle scripts run as root with the full privileges of the execution role, so uncontrolled edits are a privilege-escalation path. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/nbi-root-access.html)
- **[Network isolation]** Deploy notebook instances, training jobs, and endpoints inside a VPC with private subnets, no NAT/internet routes, and VPC endpoints for required AWS services (S3, ECR, etc.). — Prevents direct internet exfiltration paths while keeping access to needed AWS services. [doc](https://aws.amazon.com/blogs/architecture/preventing-data-exfiltration-in-machine-learning-environments-with-amazon-sagemaker-ai/)
- **[Endpoint access]** Restrict VPC endpoint policies to organization-owned resources only, and use AWS PrivateLink for inference calls instead of public endpoints. — Adds a safeguard against cross-account data movement and removes public internet exposure for prediction traffic. [doc](https://aws.amazon.com/blogs/machine-learning/secure-prediction-calls-in-amazon-sagemaker-with-aws-privatelink/)
- **[Inference endpoints]** Restrict access to intended consumers with least-privilege IAM policies, sign HTTPS requests, and configure security groups/VPC for inbound-outbound traffic control. — Prevents unauthorized invocation of deployed models and protects sensitive prediction data. [doc](https://docs.aws.amazon.com/wellarchitected/latest/machine-learning-lens/mlsec06-bp01.html)
- **[Data and artifacts]** Encrypt ML storage volumes, model artifacts, and S3 buckets using AWS KMS, and rely on the default SSL/TLS connection for SageMaker API and console requests. — Protects data and models at rest and in transit. [doc](https://aws.amazon.com/sagemaker/ai/faqs/)
- **[Governance at scale]** Use IAM condition keys or AWS Service Catalog with CloudFormation templates to prevent deployment of SageMaker resources (notebooks, training jobs) without required security controls, and use CloudWatch Events plus Lambda to detect and remediate non-compliant resources after creation. — Enforces secure configurations (for example disabling notebook root access, forcing VPC-only S3 access) both preventively and reactively across an organization. [doc](https://aws.amazon.com/blogs/security/secure-deployment-of-amazon-sagemaker-resources/)
- **[Credentials]** Store credentials and keys (access IDs, secrets) in AWS Secrets Manager rather than in configuration files or source code. — Avoids hardcoded secrets that can leak through notebooks or repos. [doc](https://aws.amazon.com/blogs/machine-learning/aws-internal-use-case-evaluating-and-adopting-amazon-sagemaker-within-aws-marketing/)

## 🛡️ Reliability
- **[Endpoint availability]** Deploy multiple instances per production endpoint across at least two subnets in different Availability Zones. — Protects the endpoint from an Availability Zone outage or single-instance failure. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/deployment-best-practices.html)
- **[High availability target]** For 99.95% availability, configure more than two copies of each inference component and set the auto scaling minimum instance count to at least two. — A single-instance or single-copy endpoint cannot tolerate instance failure without downtime. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/deployment-best-practices.html)
- **[Safe updates]** Use rolling deployments or blue/green deployment guardrails with a baking period and CloudWatch alarm-triggered auto-rollback when updating endpoints. — Limits blast radius of a bad model/config update and automatically reverts traffic if health metrics degrade. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/deployment-guardrails.html)
- **[Resource lifecycle]** Never delete a model definition, model artifacts, execution role, inference image, VPC config, KMS key, or endpoint configuration that an active endpoint still references; instead create new versions and repoint the endpoint. — Deleting in-use resources breaks SageMaker's automatic patching/instance-replacement and can render endpoints unrecoverable. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/best-practice-endpoint-security.html)
- **[Autoscaling]** Configure automatic scaling (target tracking, step, or simple scaling) on real-time endpoints with sensible min/max instance counts, and use Serverless Inference for unpredictable traffic. — Maintains availability under load spikes without manual intervention. [doc](https://docs.aws.amazon.com/wellarchitected/latest/machine-learning-lens/mlrel05-bp01.html)
- **[Large-scale training]** Checkpoint regularly (every few hours is common) when training at scale. — Hardware failures are expected at scale, and frequent checkpointing bounds the amount of lost training progress. [doc](https://aws.amazon.com/blogs/machine-learning/training-large-language-models-on-amazon-sagemaker-best-practices/)

## ⚡ Performance Efficiency
- **[Instance selection]** Use SageMaker Inference Recommender to benchmark and compare instance types/sizes (including AWS Inferentia and Graviton) before choosing what to deploy. — Identifies the most performant instance for a given model instead of guessing. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/inference-cost-optimization.html)
- **[Model optimization]** Compile and optimize models with SageMaker Neo, and consider pruning, quantization, or simplified algorithm variants. — Reduces inference latency and resource usage versus running unoptimized models on larger instances. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/inference-cost-optimization.html)
- **[Distributed training]** Use a cloud-optimized distributed training library (such as SageMaker sharded data parallelism) and enable EFA/NVIDIA GPUDirectRDMA for fast inter-machine communication at scale. — Ensures GPUs are used efficiently and inter-node communication does not bottleneck large training jobs. [doc](https://aws.amazon.com/blogs/machine-learning/training-large-language-models-on-amazon-sagemaker-best-practices/)
- **[Load testing]** Load test real-time endpoints to determine throughput limits before setting autoscaling thresholds. — Ensures scaling policies are tuned to actual observed bottlenecks rather than assumptions. [doc](https://aws.amazon.com/blogs/machine-learning/best-practices-for-load-testing-amazon-sagemaker-real-time-inference-endpoints/)

## 💰 Cost Optimization
- **[Inference option]** Match the inference option to the workload: real-time for steady low-latency traffic, serverless for spiky synchronous traffic, asynchronous for large payloads that can tolerate latency and scale to zero, and batch transform for offline bulk scoring. — Avoids paying for always-on capacity when traffic patterns don't require it. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/inference-cost-optimization.html)
- **[Endpoint consolidation]** Check CloudWatch utilization metrics for under-used endpoints and consolidate models onto multi-model or multi-container endpoints. — Improves instance utilization and return on investment versus running many low-traffic dedicated endpoints. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/inference-cost-optimization.html)
- **[Autoscaling]** Configure autoscaling for real-time and asynchronous endpoints instead of provisioning statically for peak traffic. — Removes unused capacity during low-traffic periods and reduces compute spend automatically. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/inference-cost-optimization.html)
- **[Commitment pricing]** Opt in to a SageMaker Savings Plan when usage is consistent across training, processing, notebooks, and inference. — Reduces costs versus on-demand pricing for predictable workloads. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/inference-cost-optimization.html)
- **[Idle notebook cleanup]** Use lifecycle configuration scripts to detect and stop idle notebook instances automatically. — Prevents paying for compute that is provisioned but not actively used. [doc](https://aws.amazon.com/blogs/machine-learning/secure-amazon-s3-access-for-isolated-amazon-sagemaker-notebook-instances/)

## ⚙️ Operational Excellence
- **[Production monitoring]** Use SageMaker Model Monitor to continuously track data quality, model quality, bias drift, and feature attribution drift, with alerting on threshold violations. — Enables early detection of data/model drift so you can retrain or fix upstream issues before predictions degrade. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/model-monitor.html)
- **[Endpoint updates]** Follow the documented CreateModel/CreateEndpointConfig/UpdateEndpoint procedure (and use unique resource names) rather than deleting and recreating resources in place. — Keeps endpoints eligible for automatic patching and avoids breaking dependent resources mid-update. [doc](https://docs.aws.amazon.com/sagemaker/latest/dg/best-practice-endpoint-security.html)
- **[MLOps automation]** Automate training and deployment workflows with SageMaker Pipelines, and gate promotions with rolling/blue-green deployment guardrails tied to CloudWatch alarms. — Creates a repeatable, auditable process for rapid experimentation and safe production rollout. [doc](https://aws.amazon.com/sagemaker/ai/train/)
- **[Well-Architected reviews]** Periodically evaluate ML workloads against the AWS Well-Architected Machine Learning Lens across the full ML lifecycle, not just before launch. — Surfaces design and operational issues early enough to address them cheaply. [doc](https://aws.amazon.com/blogs/architecture/introducing-the-well-architected-framework-for-machine-learning/)

# AWS Backup — Best Practices

## Common scenarios
- Centralized, policy-based backup across compute, storage, and database resources        → Operational Excellence, Reliability
- Ransomware and disaster recovery protection via immutable, cross-account/cross-Region copies        → Security, Reliability
- Meeting regulatory retention and audit requirements across accounts and Regions        → Security, Operational Excellence
- Cost-optimized long-term retention via automated cold-storage tiering        → Cost Optimization

## 🔒 Security
- **[Access control]** Apply the principle of least privilege on IAM identity-based policies so users and roles only get the backup and restore permissions their job requires. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Access control]** Set resource-based (vault access) policies on backup vaults to control which accounts, users, or roles can create, copy, or delete recovery points, independent of per-user IAM policies. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/access-control.html)
- **[Access control]** Use service control policies (SCPs) in AWS Organizations to enforce centralized, maximum-permission guardrails for backup vaults and operations across all member accounts. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Access control]** Use IAM Access Analyzer to detect AWS Backup IAM roles or vault policies that grant access to external accounts, root users, or anonymous principals. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Encryption]** Encrypt every backup vault with AWS KMS, using a customer managed key where you need independent key policies from the key that protects the source resource. — [doc](https://aws.amazon.com/documentation-overview/backup/)
- **[Encryption]** For cross-account backup copies, share the source customer managed KMS key with the destination account, since AWS managed keys cannot be shared across accounts. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/create-cross-account-backup.html)
- **[Encryption]** Evaluate multi-Region KMS keys to simplify key management when backups must be copied and decrypted in another AWS Region for disaster recovery. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Immutability]** Apply AWS Backup Vault Lock in compliance mode so recovery points cannot be deleted or have their retention shortened by any user, including the account root, during the required retention period. — [doc](https://aws.amazon.com/blogs/storage/enhance-the-security-posture-of-your-backups-with-aws-backup-vault-lock/)
- **[Immutability]** Account for the mandatory 72-hour cooling-off period before a Vault Lock configuration becomes immutable, and set `ChangeableForDays` to at least 3. — [doc](https://docs.aws.amazon.com/cdk/api/v2/docs/aws-cdk-lib.aws_backup.CfnBackupVault.LockConfigurationTypeProperty.html)
- **[Ransomware resilience]** Use logically air-gapped vaults to store immutable, encrypted backup copies isolated in an AWS Backup–owned account, separate from your workload accounts. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/logicallyairgappedvault.html)
- **[Ransomware resilience]** Pair logically air-gapped vaults with Multi-party approval so backups remain recoverable even if the vault-owning or workload account becomes inaccessible or compromised. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/logicallyairgappedvault.html)
- **[Cross-account DR]** Copy backups to a separate AWS account within your AWS Organization so a compromised or deleted source backup can still be recovered from the destination account. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/create-cross-account-backup.html)
- **[Cross-Region DR]** Copy backups across AWS Regions to meet business continuity or compliance requirements to store recovery points a minimum distance from production data. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/cross-region-backup.html)
- **[Governance]** Use legal holds to prevent deletion of specific recovery points for e-discovery or litigation, independent of the configured retention period. — [doc](https://aws.amazon.com/backup/features/)
- **[Detective controls]** Enable AWS Security Hub with the AWS Foundational Security Best Practices standard to get detective controls covering backup vault and plan misconfigurations. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Incident response]** Incorporate AWS Backup into incident response and forensics playbooks, for example using instance- and volume-level backups to support automated forensic disk collection. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)

## 🛡️ Reliability
- **[Strategy]** Define a backup strategy that maps each workload's acceptable RTO and RPO to a concrete backup cadence, retention, and recovery pattern (point-in-time, file-level, volume-level, instance-level). — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[DR integration]** Incorporate AWS Backup procedures into your organization's disaster recovery and business continuity plans, and rehearse them so staff are familiar with the failover process. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/security-considerations.html)
- **[Automation]** Define backup requirements as backup plans and apply them to resources via tag-based resource assignment, rather than configuring backups resource-by-resource. — [doc](https://aws.amazon.com/documentation-overview/backup/)
- **[Automation at scale]** Use AWS Organizations backup policies to centrally enforce backup plans across all accounts in your organization instead of relying on per-account configuration. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Restore testing]** Create restore testing plans that automatically run scheduled restore jobs against selected or random recovery points, and use restore testing validation to confirm the restored data is usable. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/restore-testing.html)
- **[Restore testing]** Test cross-account, cross-Region restore paths regularly (for example DR vault to source vault, each with different customer managed KMS keys) since encryption or permission assumptions that go untested often turn out to be wrong. — [doc](https://docs.aws.amazon.com/prescriptive-guidance/latest/security-best-practices/test.html)
- **[Restore testing]** Keep the number of recovery paths and patterns you rely on limited and feasible so each one can realistically be tested on a routine basis. — [doc](https://docs.aws.amazon.com/prescriptive-guidance/latest/security-best-practices/test.html)
- **[Tag retention]** Use AWS Backup events to trigger tag replication onto resources created by restore jobs, since tags are not copied from recovery points to restored resources by default. — [doc](https://aws.amazon.com/blogs/storage/how-to-retain-resource-tags-in-aws-backup-restore-jobs/)
- **[Monitoring]** Monitor backup, copy, and restore job status with Amazon CloudWatch metrics and alarms so failures are caught immediately rather than discovered when a restore is needed. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Monitoring]** Subscribe to AWS Backup events through Amazon EventBridge and Amazon SNS to drive automated alerting and escalation workflows for failed jobs. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)

## 💰 Cost Optimization
- **[Lifecycle management]** Configure lifecycle policies to automatically transition recovery points to cold storage after a defined age, reducing storage cost for data accessed infrequently. — [doc](https://docs.aws.amazon.com/help-panel/aws-backup/latest/helppanel/edit-recovery-point-lifecycle.html)
- **[Lifecycle management]** Set the retention period at least 90 days beyond the cold-storage transition point, since backups moved to cold storage must remain there a minimum of 90 days. — [doc](https://docs.aws.amazon.com/aws-backup/latest/APIReference/API_CalculatedLifecycle.html)
- **[Restore testing cost]** Scope restore testing plans to only your most critical resources and align test schedules with retention windows to limit the storage and compute cost of repeated test restores. — [doc](https://aws.amazon.com/blogs/storage/implementing-restore-testing-for-recovery-validation-using-aws-backup/)
- **[Ransomware protection cost]** Use logically air-gapped vault primary backups when a single immutable copy meets your resilience needs, avoiding the storage cost of maintaining multiple redundant copies while still getting Vault Lock protection. — [doc](https://aws.amazon.com/blogs/storage/streamline-ransomware-protection-while-reducing-cost-with-logically-air-gapped-vault-primary-backup/)
- **[Recovery path pruning]** Limit the number of cross-account/cross-Region recovery paths you maintain to what you can realistically test, avoiding wasted storage on redundant, unverified copies. — [doc](https://docs.aws.amazon.com/prescriptive-guidance/latest/security-best-practices/test.html)

## ⚙️ Operational Excellence
- **[Compliance automation]** Use AWS Backup Audit Manager to define frameworks of controls (backup frequency, retention, encryption) and automatically detect resources or activity that violate them. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/aws-backup-audit-manager.html)
- **[Compliance reporting]** Generate daily and on-demand AWS Backup Audit Manager reports as evidence of compliance for auditors and regulators, and import findings into AWS Audit Manager for a unified compliance view. — [doc](https://docs.aws.amazon.com/aws-backup/latest/devguide/whatisbackup.html)
- **[IaC governance]** Deploy AWS Backup Audit Manager frameworks and controls via AWS CloudFormation to keep backup governance consistent and version-controlled across environments. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Centralized delegation]** Delegate backup policy management and cross-account monitoring to a dedicated backup administrator account in AWS Organizations instead of requiring access to the management account for daily operations. — [doc](https://aws.amazon.com/backup/features/)
- **[Auditability]** Use AWS CloudTrail to record AWS Backup API calls (time, source IP, user, account) for investigation and accountability. — [doc](https://aws.amazon.com/blogs/security/top-10-security-best-practices-for-securing-backups-in-aws/)
- **[Cross-account visibility]** Aggregate daily backup activity across multiple accounts and Regions into centralized dashboards to maintain organization-wide operational awareness. — [doc](https://aws.amazon.com/blogs/storage/obtain-aggregated-daily-cross-account-multi-region-aws-backup-reporting/)

# AWS License Manager — Best Practices

## Common scenarios
- Governing BYOL usage limits on EC2 launches for vendor software (Microsoft, SAP, Oracle, IBM)        → Security, Cost Optimization
- Tracking software license inventory and usage across accounts, Regions, and on-premises servers        → Operational Excellence, Cost Optimization
- Preventing non-compliant instance launches and preparing for vendor license audits        → Security, Operational Excellence
- Distributing and tracking seller-issued or Marketplace license entitlements across an organization        → Reliability, Operational Excellence

## 🔒 Security
- **[least privilege]** Create IAM policies that explicitly grant only the License Manager actions each administrator or user needs, since by default users have no permissions on License Manager resources — this avoids overly broad access to license configurations and entitlements. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/identity-access-management.html)
- **[MFA]** Require multi-factor authentication on accounts that manage License Manager resources and set up individual IAM or IAM Identity Center users instead of sharing root or broad credentials — this limits each user to only the permissions needed for their job. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/data-protection.html)
- **[transport security]** Require clients to use TLS 1.2 or later (TLS 1.3 recommended) with cipher suites supporting perfect forward secrecy (such as ECDHE) when calling License Manager APIs — this protects license and entitlement data in transit. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/infrastructure-security.html)
- **[sensitive data]** Never place confidential or sensitive information, such as customer email addresses, into License Manager tags or free-form text fields — this data can surface in billing or diagnostic logs. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/data-protection.html)
- **[service-linked roles]** Review the scope of AWS managed policies attached to License Manager service-linked roles (for example AWSLicenseManagerUserSubscriptionsServiceRolePolicy) before enabling cross-service features like user-based subscriptions or Marketplace grant distribution — these roles carry specific EC2, SSM, and Secrets Manager permissions scoped to License Manager operations. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/security-iam-awsmanpol.html)
- **[hard limits]** Enable the "Enforce license limit" (hard limit) option on license configurations for agreements where overage is unacceptable, so License Manager blocks non-compliant instance launches outright rather than only alerting after the fact. [doc](https://aws.amazon.com/blogs/mt/mechanisms-to-govern-license-usage-with-aws-license-manager/)
- **[compliance validation]** Use AWS Artifact to download third-party audit reports relevant to License Manager and confirm which compliance programs it falls under before relying on it for regulated licensing workloads. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/compliance-validation.html)

## 🛡️ Reliability
- **[license release handling]** Decide deliberately whether stopped instances should continue consuming licenses by configuring the "Include stopped instances" rule — by default a license is released back to the pool when an instance stops, which may not match agreements that require licenses to remain allocated regardless of running state. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/license-rules.html)
- **[multi-Region tracking]** Create license configurations in every Region where licensed workloads run and track them centrally, since License Manager enforces limits and counts entitlements per Region — a multi-Region tracking solution avoids blind spots in usage visibility. [doc](https://aws.amazon.com/blogs/mt/track-licenses-across-multiple-aws-regions/)
- **[organization-wide governance]** Use License asset groups to consolidate license discovery and enforcement across multiple AWS Regions and accounts within AWS Organizations instead of managing configurations independently per account — this reduces the risk of inconsistent or missed enforcement during scale-out or M&A integration. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/license-manager-overview.html)
- **[quota awareness]** Review License Manager service quotas (license configurations per account, associations per resource, grants per license, accounts per organization) before large-scale rollouts, and request increases for adjustable quotas ahead of time to avoid hitting limits during deployment. [doc](https://docs.aws.amazon.com/general/latest/gr/licensemanager.html)
- **[attach rules at the source]** Attach license configurations to AMIs, EC2 launch templates, or CloudFormation templates used as the standard deployment mechanism, rather than relying on manual per-instance tracking — this ensures every deployment from that source is consistently and automatically tracked. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/license-rules.html)

## ⚙️ Operational Excellence
- **[centralized inventory]** Use AWS Systems Manager Inventory integration with License Manager to discover and track on-premises applications alongside cloud resources — this gives a single view of license consumption across hybrid environments. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/license-manager.html)
- **[monitoring]** Set up Amazon CloudWatch alarms on License Manager usage metrics to get near real-time notice when license consumption approaches configured limits, rather than discovering overages only during a vendor audit. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/monitoring-overview.html)
- **[audit trail]** Enable an AWS CloudTrail trail covering License Manager API calls (CreateLicenseConfiguration, ListUsageForLicenseConfiguration, DeleteLicenseConfiguration, and others) delivered to Amazon S3 — this provides the identity, source IP, and timing detail needed to prepare for vendor audits and investigate configuration changes. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/logging-using-cloudtrail.html)
- **[event-driven automation]** Match License Manager CloudTrail events in Amazon EventBridge (source `aws.license-manager`) to automate notifications or remediation workflows when specific license API actions occur. [doc](https://docs.aws.amazon.com/eventbridge/latest/ref/events-ref-license-manager.html)
- **[soft-limit alerting]** Configure Amazon SNS notifications for soft-limit violations so administrators and end users are alerted by email or text when usage exceeds agreed thresholds, even when launches are not blocked outright. [doc](https://aws.amazon.com/license-manager/features/)
- **[Service Catalog governance]** Add license rules to AWS Service Catalog products so that only approved, license-compliant IT services can be provisioned across accounts by end users. [doc](https://docs.aws.amazon.com/whitepapers/latest/aws-overview/management-governance.html)

## 💰 Cost Optimization
- **[BYOL vs license-included]** Use License Manager's license type conversion capability to switch workloads between AWS-provided licensing and Bring Your Own License (BYOL) without redeploying, choosing whichever model is more cost-effective as agreements or workload patterns change. [doc](https://docs.aws.amazon.com/license-manager/latest/userguide/license-manager.html)
- **[reuse existing entitlements]** Repurpose existing on-premises license inventory for cloud resources through BYOL rather than purchasing new AWS-provided licenses, using License Manager to track and enforce the repurposed entitlements. [doc](https://aws.amazon.com/blogs/mt/mechanisms-to-govern-license-usage-with-aws-license-manager/)
- **[license recommendations]** Review AWS Compute Optimizer's commercial software license recommendations for EC2 instances to identify oversized license editions and right-size license consumption based on actual utilization data. [doc](https://docs.aws.amazon.com/compute-optimizer/latest/ug/view-license-recommendations.html)
- **[reallocate underutilized licenses]** Use License Manager's centralized inventory and usage tracking to identify underutilized licenses across accounts (particularly after M&A or organizational consolidation) and reallocate them instead of purchasing additional entitlements. [doc](https://aws.amazon.com/blogs/mt/solving-license-management-challenges-during-mergers-acquisitions-with-aws-license-manager/)
